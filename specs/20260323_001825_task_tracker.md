# App Specification: Task Tracker

## 1. Overview
The Task Tracker is a comprehensive application designed to help users manage their personal or professional tasks efficiently. It includes features such as task categorization, setting deadlines, scheduling reminders, and collaboration with others. The app aims to provide a user-friendly interface for creating, reading, updating, and deleting tasks while ensuring secure authentication and authorization.

## 2. App Type & Platform
- **App Type:** Task Tracker
- **Platform:** Web-based application

## 3. Core Features
1. **Task Management:**
   - Create new tasks with categories, deadlines, and descriptions.
   - View all tasks with filters for categories and deadlines.
   - Edit existing tasks.
   - Delete tasks.

2. **Reminders:**
   - Set reminders for tasks to notify users via email or in-app notifications.

3. **Collaboration:**
   - Share tasks with other users.
   - Assign tasks to team members.
   - Comment on tasks and collaborate in real-time.

4. **User Authentication:**
   - Register new users.
   - Login and logout functionality.
   - Secure user profiles.

- **Data Flow Description:**
  - The front-end sends requests to the API endpoints defined in the `api/` directory.
  - The back-end processes these requests using Next.js API routes and interacts with the PostgreSQL database via TypeORM.
  - Responses from the back-end are sent back to the front-end, which updates the user interface accordingly.

## 6. Data Models
- **User Model:**
  ```javascript
  import { Entity, PrimaryGeneratedColumn, Column } from 'typeorm';

  @Entity()
  export class User {
    @PrimaryGeneratedColumn()
    id: number;

    @Column()
    username: string;

    @Column()
    email: string;

    @Column()
    password: string;
  }
  ```

- **Task Model:**
  ```javascript
  import { Entity, PrimaryGeneratedColumn, Column, ManyToOne } from 'typeorm';
  import { User } from './User';

  @Entity()
  export class Task {
    @PrimaryGeneratedColumn()
    id: number;

    @Column()
    title: string;

    @Column()
    description: string;

    @Column()
    category: string;

    @Column()
    deadline: Date;

    @ManyToOne(() => User, user => user.tasks)
    user: User;
  }
  ```

## 7. API Design
- **User Authentication:**
  - `POST /api/auth/register`: Register a new user.
    ```json
    {
      "username": "string",
      "email": "string",
      "password": "string"
    }
    ```
  - `POST /api/auth/login`: Authenticate a user and return a JWT token.
    ```json
    {
      "email": "string",
      "password": "string"
    }
    ```
  - `GET /api/auth/profile`: Get the current user's profile (protected by JWT).

- **Task Management:**
  - `POST /api/tasks`: Create a new task.
    ```json
    {
      "title": "string",
      "description": "string",
      "category": "string",
      "deadline": "date"
    }
    ```
  - `GET /api/tasks`: Retrieve all tasks (with optional filters like category, deadline).
  - `GET /api/tasks/:id`: Retrieve a specific task by ID.
  - `PUT /api/tasks/:id`: Update an existing task.
    ```json
    {
      "title": "string",
      "description": "string",
      "category": "string",
      "deadline": "date"
    }
    ```
  - `DELETE /api/tasks/:id`: Delete a task.

## 8. User Authentication (if applicable)
- **JWT Implementation:**
  - The app uses JSON Web Tokens (JWT) for user authentication and authorization.
  - JWT tokens are generated upon successful login and must be included in the Authorization header for protected routes.

## 9. Development Environment Setup
1. **Front-end:**
   ```bash
   npx create-next-app@latest task-tracker
   cd task-tracker
   npm install react react-dom next
   ```

2. **Back-end:**
   ```bash
   npm init -y
   npm install express typeorm pg jsonwebtoken bcryptjs
   ```

3. **Database Setup:**
   - Install PostgreSQL and create a new database.
   - Configure TypeORM to connect to the PostgreSQL database.

4. **Environment Variables:**
   - Create a `.env` file in the root directory with the following variables:
     ```plaintext
     DATABASE_URL=postgres://username:password@localhost:5432/task-tracker
     JWT_SECRET=your_jwt_secret_key
     ```

## 10. Build & Deployment
- **Local Development:**
  - Start the front-end development server:
    ```bash
    npm run dev
    ```
  - Start the back-end server:
    ```bash
    node server.js
    ```

- **Deployment on VPS Server:**
  - Install Node.js and PostgreSQL on your VPS.
  - Clone the repository to your VPS.
  - Set up environment variables in a `.env` file.
  - Build the front-end application:
    ```bash
    npm run build
    ```
  - Start the back-end server:
    ```bash
    node server.js
    ```
